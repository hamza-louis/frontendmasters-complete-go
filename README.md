- The compiler needs to look into package with the name main and a main function
- The compiler will not compile the code if you have unused imports in a Go file
- The purpose of the package `main` in a Go application, is to create executable binaries
- Unused variables aren't allowed in Go
- Printf needs a new line specifier ("\n") where Println doesn't
- To allow the compiler to infer a variable's type, you need to use `:=` in the value assignemnt
